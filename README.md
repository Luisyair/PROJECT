## 🚀 Ejecución del Proyecto en Entorno Local

Este proyecto puede ser ejecutado localmente para propósitos de desarrollo, pruebas o demostraciones. A continuación, se detallan los pasos necesarios para ponerlo en marcha en tu entorno local.

### ✅ Requisitos Previos

Antes de comenzar, asegúrate de cumplir con los siguientes requisitos:

* Tener instalado **Node.js** en su versión 14 o superior.
* Contar con una terminal o línea de comandos (como Terminal en macOS/Linux o CMD/PowerShell en Windows).
* Tener instalado un gestor de paquetes como **npm** o **yarn**.

Puedes verificar la versión de Node.js instalada ejecutando:

```bash
node -v
```

Si no tienes Node.js instalado, puedes descargarlo desde: [https://nodejs.org](https://nodejs.org)

---

### 📦 Paso 1: Clonar el Repositorio

Para obtener una copia local del proyecto, abre una terminal y ejecuta el siguiente comando:

```bash
git clone https://github.com/jotredev/dashboard-store-v1.git
```

Este comando creará una nueva carpeta con el contenido del proyecto.

---

### 📁 Paso 2: Acceder al Directorio del Proyecto

Una vez clonado el repositorio, accede al directorio del proyecto:

```bash
cd dashboard-store
```

---

### 📥 Paso 3: Instalar Dependencias

Dentro del directorio del proyecto, instala todas las dependencias necesarias. Puedes hacerlo usando **npm** o **yarn**, según tu preferencia:

```bash
npm install
# o
yarn
```

Esto descargará e instalará todos los paquetes definidos en el archivo `package.json`, que son necesarios para que la aplicación funcione correctamente.

---

### ▶️ Paso 4: Iniciar el Servidor de Desarrollo

Una vez instaladas las dependencias, ya puedes ejecutar el proyecto en modo desarrollo. Utiliza uno de los siguientes comandos:

```bash
npm run dev
# o
yarn dev
```

Esto iniciará un servidor local (generalmente en `http://localhost:3000`) desde donde podrás ver y trabajar con la aplicación en tiempo real.

---

### 🧪 Verificación

Si todo ha salido correctamente, deberías ver un mensaje en la terminal indicando que el servidor está activo, junto con la URL en la que puedes acceder a la aplicación. Abre esa URL en tu navegador para comprobar que todo funciona como se espera.

---

### ❓ Soporte

Si encuentras algún problema durante el proceso, asegúrate de revisar:

* La salida de la terminal para mensajes de error.
* Que estés utilizando versiones compatibles de Node.js y los gestores de paquetes.
* La documentación del proyecto en el repositorio de GitHub.
